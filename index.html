Overview

Changelog: Kata_web 1.0.2 (Bug Fixes and Improvements)

Overview

Kata_web 1.0.2 is a maintenance release that focuses on bug fixes and improvements to make the script more robust and user-friendly. This version addresses issues related to file handling, error management, and logging, ensuring a smoother experience for users.

Bug Fixes

1. FileNotFoundError Handling

  • Fixed the issue where the script would crash if a file (e.g., katago-v1.15.3-eigen-linux-x64.zip) was missing or already deleted.
  • Added checks using os.path.exists() before attempting to delete files.

2. Error Handling

  • Wrapped file operations in try-except blocks to handle errors gracefully.
  • The script now continues running even if one step fails, making it more robust.

3. Improved Logging

  • Added more descriptive print statements to help debug issues.
  • Each step now logs its progress and any errors encountered.

Improvements

1. Graceful Degradation

  • If a step fails (e.g., downloading or extracting a file), the script logs the error and continues with the next step.

2. User-Friendly Messages

  • Added clear and informative messages for each step, making it easier to understand what the script is doing.

3. Code Cleanup

  • Removed redundant code and improved readability.

Features

1. Dependency Setup

  • libzip5 and libssl1.1 are downloaded and extracted to ensure compatibility with KataGo.
  • The LD_LIBRARY_PATH environment variable is set to include the extracted libraries.

2. KataGo Setup

  • KataGo (Eigen version) is downloaded and extracted.
  • The KataGo binary is made executable.

3. KataGo Model Download

  • The KataGo b18 model is downloaded and extracted for use by the bot.

4. gtp2ogs Setup

  • gtp2ogs is downloaded and made executable. This tool acts as a bridge between KataGo and OGS.

5. Configuration Updates

  • default_gtp.cfg is updated with the following changes:
    • Enabled logSearchInfo.
    • Enabled ogsChatToStderr.
    • Set maxTime to 1.0 seconds.
    • Enabled ponderingEnabled.
    • Updated rules to Tromp-Taylor with friendly pass allowed.

6. kata_speed.json5 Configuration

  • A configuration file (kata_speed.json5) is generated with the following settings:
    • Blacklist: Blocks specific users from challenging the bot.
    • Whitelist: Allows specific users to challenge the bot even when it‚Äôs busy.
    • Ranked and Unranked Games: Both are allowed.
    • Board Sizes: Supports 9x9, 13x13, and 19x19 boards.
    • Correspondence Games: Disabled.
    • Bot Behavior: Sends PV data and chat messages.

7. Bot Execution

  • The bot is launched using gtp2ogs with the configured API key, KataGo binary, and model.

Technical Details

Dependencies

  • libzip5: Required for handling ZIP files.
  • libssl1.1: Required for secure communication.
  • KataGo (Eigen version): The Go engine used for gameplay.
  • gtp2ogs: The bridge between KataGo and OGS.

Configuration Files

  • default_gtp.cfg: Contains KataGo‚Äôs GTP configuration.
  • kata_speed.json5: Contains the bot‚Äôs OGS-specific settings.

Script Workflow

  1. Downloads and extracts dependencies.
  2. Downloads and sets up KataGo.
  3. Downloads the KataGo model.
  4. Downloads and sets up gtp2ogs.
  5. Updates default_gtp.cfg with custom settings.
  6. Generates kata_speed.json5 for OGS configuration.
  7. Runs the bot using gtp2ogs.

Known Issues

  • Correspondence Games: Disabled in this version. Only live games are supported.
  • Performance: The bot is configured for low visit counts (7 visits) due to resource limitations. Upgrading to a GPU backend (e.g., CUDA or TensorRT) is recommended for stronger performance.

Future Improvements

  • GPU Support: Add support for CUDA or TensorRT backends for stronger performance.
  • Advanced Analytics: Provide detailed game analysis and visualizations for players.
  • Multi-Language Support: Add support for multiple languages in bot greetings and farewells.
  • Scalability: Increase the number of concurrent games the bot can handle.

  • gtp2ogs: online-go
